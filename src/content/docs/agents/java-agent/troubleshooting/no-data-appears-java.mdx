<div>
  <h2>Problem</h2>
  <p>
    After generating traffic for your app and waiting a few minutes for data to
    appear in the New Relic UI, your Java app isn't reporting data.
  </p>
  <h2>Solution</h2>
  <p>
    After sending a request to your web application, data should appear in the
    APM UI within two to three minutes. If no data appears after a few minutes:
  </p>
  <ol>
    <li>Restart your web server.</li>
    <li>
      Use{' '}
      <a href="/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics">
        New Relic Diagnostics
      </a>{' '}
      to try to automatically identify the issue.
    </li>
    <li>
      <p>
        Verify the <code>newrelic.jar</code> file has been installed correctly
        and that the <code>-javaagent</code> argument has been passed correctly
        into the JVM:
      </p>
      <ol>
        <li>Start your web server if it isn't already running.</li>
        <li>
          <p>In your terminal or console, run the appropriate command:</p>
          <p>Linux or macOS:</p>
          <pre>ps -ef | grep java</pre>
          <p>Windows:</p>
          <pre>
            wmic process where "name='java.exe'" get ProcessID, Commandline
            /format:list
          </pre>
        </li>
        <li>
          <p>
            Find the process for the service you're trying to monitor and see if
            the <code>-javaagent</code> argument is present:
          </p>
          <ul>
            <li>
              If you see a <code>-javaagent</code> argument: The issue is either
              in agent startup, or the agent is unable to communicate through
              your firewall to{' '}
              <a href="/docs/using-new-relic/cross-product-functions/install-configure/networks">
                our collector's IP range
              </a>
              .
            </li>
            <li>
              If you don't see a <code>-javaagent</code> argument: The argument
              wasn't passsed to your JVM. Check your startup scripts or other
              resources to ensure there's nothing preventing it from being
              passed. For more info about how to pass the argument for your
              environment, see{' '}
              <a href="/docs/agents/java-agent/installation/include-java-agent-jvm-argument">
                Include the Java agent with a JVM argument
              </a>
              .
            </li>
          </ul>
        </li>
        <li>If niether solution works, keep reading.</li>
      </ol>
    </li>
    <li>
      <p>
        Verify that your <code>newrelic.jar</code> and <code>newrelic.yml</code>{' '}
        files are in the same{' '}
        <a href="/docs/agents/manage-apm-agents/troubleshooting/find-agent-root-directory#java-agent">
          directory
        </a>
        . If you have specified a non-default location for the config file using
        the{' '}
        <code>
          <a href="/docs/agents/java-agent/configuration/java-agent-configuration-config-file#newrelic-config-file">
            newrelic.config.file
          </a>{' '}
        </code>
        system property, ensure that the config file does exist in that location
        and is correctly formatted.
      </p>
    </li>
    <li>
      <a href="/docs/agents/java-agent/troubleshooting/generating-logs-troubleshooting">
        Generate <code>finest</code> level logs
      </a>
      , and check the logs for errors. In the event no log file can be found,
      see{' '}
      <a href="/docs/agents/java-agent/troubleshooting/no-log-file-java">
        No log file (Java)
      </a>
      .
    </li>
    <li>
      Check the application server logs for errors related to the new relic java
      agent. You can see the troubleshooting pages{' '}
      <a href="https://docs.newrelic.com/docs/agents/java-agent/troubleshooting/error-bootstrapping-new-relic-java-agent">
        Error Bootstrap New Relic Java Agent
      </a>{' '}
      and{' '}
      <a href="https://docs.newrelic.com/docs/agents/java-agent/troubleshooting/errors-starting-java-app-server">
        Errors Java App Server
      </a>{' '}
      for examples of observable errors in application server logs that prevent
      the java agent data from reporting.
    </li>
    <li>
      <p>
        Verify that your app is reporting to the expected name: In your New
        Relic log files, search for <code>"reporting to"</code>, then select the
        link in the message. For example:
      </p>
      <pre>
        {'{'}"message":"Reporting to: https://rpm.newrelic.com/accounts/
        <var>000</var>/applications/<var>000000</var>"{'}'}
      </pre>
      <p>
        If you are reporting to{' '}
        <a href="/docs/apm/new-relic-apm/installation-configuration/using-multiple-names-app">
          multiple application names
        </a>
        , look for multiple lines with this message.
      </p>
    </li>
    <li>
      If the message link does not go to the expected application, ensure that
      your app name settings are the same in both <code>newrelic.yml</code> and
      in the <code>web.xml</code> <code>display-name</code> setting.
    </li>
  </ol>
</div>
