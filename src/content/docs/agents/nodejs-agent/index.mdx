Pinpoint and solve issues down to the line of code with Node monitoring from New Relic. With features like [service maps](/docs/data-analysis/service-maps/get-started/introduction-service-maps) and [error analytics](/docs/apm/applications-menu/error-analytics/error-analytics-explore-events-behind-errors), New Relic for Node.js allows you to get the full picture of your app environment.

*   Monitor your Node.js data from detailed dashboards in [New Relic APM](/docs/apm/new-relic-apm/getting-started/introduction-new-relic-apm), [New Relic Browser](/docs/browser/new-relic-browser/getting-started/introduction-new-relic-browser) and other [New Relic products](https://newrelic.com/products).
*   Query, visualize, and share your data with [New Relic Insights](/docs/insights/using-insights-ui/getting-started/introduction-new-relic-insights).
*   Enable [distributed tracing](/docs/apm/distributed-tracing/getting-started/introduction-distributed-tracing) to get visibility into modern distributed architectures.

To keep your agent up to date and ensure you have access to the latest features, see the [Node.js agent release notes](/docs/release-notes/agent-release-notes/nodejs-release-notes).
