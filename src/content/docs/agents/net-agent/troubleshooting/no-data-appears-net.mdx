<div>
  <h2>Problem</h2>
  <p>
    After generating traffic for your app and waiting five minutes, none of your
    .NET agent data appears in the New Relic UI.
  </p>
  <h2>Solution</h2>
  <p>The absence of data may occur for several reasons.</p>
  <ul>
    <li>
      <b>Common causes</b> for missing data: See the{' '}
      <a href="#basic-solutions">basic troubleshooting solutions</a>.
    </li>
    <li>
      <b>Other situations:</b> If the basic solutions do not solve the problem,
      the lack of data may be more intrinsic to your system. For more
      information, see the{' '}
      <a href="#advanced-solutions">advanced troubleshooting solutions</a>.
    </li>
    <li>
      <b>Microsoft Azure users:</b> See the{' '}
      <a href="/docs/agents/net-agent/azure-troubleshooting">
        Azure troubleshooting solutions
      </a>
      .
    </li>
    <li>
      <b>No Browser data appears:</b> See the{' '}
      <a href="/docs/agents/net-agent/troubleshooting/no-page-load-timing-data-appears-net">
        troubleshooting tips when using the Browser agent
      </a>
      .
    </li>
  </ul>
  <h3 id="basic-solutions">Basic solutions</h3>
  <p>
    To troubleshoot the most common reasons for missing data, try these
    solutions as applicable.
  </p>
  <table>
    <thead>
      <tr>
        <th>
          <b>Basic solutions</b>
        </th>
        <th>
          <b>Comments</b>
        </th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>Check app and framework for compatibility with agent</td>
        <td>
          <p>
            <a href="https://docs.newrelic.com/docs/agents/net-agent/getting-started/compatibility-requirements-net-framework-agent">
              .NET Framework compatibility and requirements
            </a>
          </p>
          <p>or</p>
          <p>
            <a href="https://docs.newrelic.com/docs/agents/net-agent/getting-started/compatibility-requirements-net-core-20-agent">
              .NET Core compatibility and requirements
            </a>
          </p>
        </td>
      </tr>
      <tr>
        <td>Configure for short-lived .NET process</td>
        <td>
          <p>
            If you have a short-lived (below 60 seconds) process, it may not
            report data. For more information, see{' '}
            <a href="/docs/agents/net-agent/troubleshooting/monitor-short-lived-net-processes">
              Monitor short-lived processes
            </a>
            .
          </p>
        </td>
      </tr>
      <tr>
        <td>
          <b>.NET Core:</b> Check Kestrel requirements.
        </td>
        <td>
          <p>
            Make sure you are using a{' '}
            <a href="/docs/agents/net-agent/getting-started/compatibility-requirements-net-core-20-agent#app-web-servers">
              Kestrel app/web server
            </a>{' '}
            supported by New Relic. Otherwise, you may need to ensure your .NET
            agent has{' '}
            <a href="/docs/agents/net-agent/getting-started/compatibility-requirements-net-framework-agent#app-web-servers">
              .NET Framework support
            </a>{' '}
            enabled.
          </p>
        </td>
      </tr>
      <tr>
        <td>
          <b>.NET Core:</b> Check environment variables.
        </td>
        <td>
          <p>
            Make sure you have configured all the{' '}
            <a href="https://docs.newrelic.com/docs/agents/net-agent/installation/install-net-core-20-agent-linux#environment_setup">
              required environment variables
            </a>
            , then restart the agent.
          </p>
        </td>
      </tr>
      <tr>
        <td>
          <b>.NET Framework:</b> Restart IIS.
        </td>
        <td>
          <p>
            If your app is not receiving traffic, no data will be reported to
            New Relic. To restart IIS:
          </p>
          <ol>
            <li>
              From a command line, run <code>IISRESET</code>.
            </li>
            <li>Verify that traffic is being sent to your application.</li>
          </ol>
        </td>
      </tr>
      <tr>
        <td>Run New Relic Diagnostics.</td>
        <td>
          To try to automatically identify the issue, use{' '}
          <a href="/docs/agents/manage-apm-agents/troubleshooting/new-relic-diagnostics">
            New Relic Diagnostics
          </a>
          .
        </td>
      </tr>
      <tr>
        <td>
          Check for{' '}
          <a href="/docs/agents/net-agent/troubleshooting/profiler-conflicts">
            profiler conflicts
          </a>
          .
        </td>
        <td>
          If you are using Microsoft's System Center Operations Manager (SCOM),
          follow the troubleshooting procedures to{' '}
          <a href="/docs/dotnet/dotnet-kb-104">
            resolve .NET and SCOM conflicts
          </a>
          .
        </td>
      </tr>
      <tr>
        <td>Review log files.</td>
        <td>
          If{' '}
          <a href="/docs/agents/net-agent/troubleshooting/generate-logs-troubleshooting-net">
            log files
          </a>{' '}
          exist, open them and review for errors.
        </td>
      </tr>
    </tbody>
  </table>
  <h3 id="advanced-solutions">Advanced solutions</h3>
  <p>
    To remedy more complex issues when no data appears, try these solutions as
    applicable:
  </p>
  <table>
    <thead>
      <tr>
        <th>
          <b>Advanced solutions</b>
        </th>
        <th>
          <b>Comments</b>
        </th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>
          <b>.NET Framework:</b> Check Registry permissions.
        </td>
        <td>
          Check for a{' '}
          <a href="/docs/agents/net-agent/troubleshooting/no-data-registry-key-permission-issues">
            registry key permissions error
          </a>
          .
        </td>
      </tr>
      <tr>
        <td>
          <b>.NET Framework:</b> Run Windows Event Viewer.
        </td>
        <td>
          <p>If applicable:</p>
          <ol>
            <li>
              Run the Windows Event Viewer, and look for application events from
              the New Relic .NET Profiler.
            </li>
            <li>
              Search for messages whose source is the New Relic .NET Profiler or
              New Relic .NET agent.
            </li>
            <li>
              If there are no New Relic event messages, look for messages from
              the .NET Runtime.
            </li>
          </ol>
          <p>
            For more information, see{' '}
            <a href="/docs/dotnet/dotnet-kb-102">
              No data or event log: CoCreate errors
            </a>
            .
          </p>
        </td>
      </tr>
      <tr>
        <td>Get debug logs.</td>
        <td>
          If your logs did not show errors, increase the logging level
          temporarily to{' '}
          <a href="/docs/agents/net-agent/troubleshooting/generating-logs-troubleshooting#logs">
            collect <code>debug</code> logs
          </a>
          .
        </td>
      </tr>
      <tr>
        <td>Check proxy settings.</td>
        <td>
          Ensure your{' '}
          <a href="/docs/dotnet/dotnet-agent-configuration#service">
            proxy settings
          </a>{' '}
          are correct and that your settings include a host attribute.
        </td>
      </tr>
      <tr>
        <td>Get additional help.</td>
        <td>
          If data still does not appear, get support at{' '}
          <a href="https://support.newrelic.com">support.newrelic.com</a>.
        </td>
      </tr>
    </tbody>
  </table>
</div>
