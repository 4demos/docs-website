<div>
  <h2>Syntax</h2>
  <pre>
    NewRelic.Api.Agent.NewRelic.SetUserParameters(<em>string</em> $user_value,{' '}
    <em>string</em> $account_value, <em>string</em> $product_value)
  </pre>
  Create user-related custom attributes. AddCustomParameter() is more flexible.{' '}
  <h2>Requirements</h2>
  <p>Compatible with all agent versions.</p>
  <p>
    Must be called inside a{' '}
    <a href="/docs/accounts-partnerships/education/getting-started-new-relic/glossary#transaction">
      transaction
    </a>
    .
  </p>
  <h2>Description</h2>
  <div className="callout-tip">
    <p>
      This call only allows you to assign values to pre-existing keys. For a
      more flexible method to create key/value pairs, use{' '}
      <a href="/docs/agents/net-agent/net-agent-api/addcustomparameter-net-agent">
        <code>AddCustomParameter()</code>
      </a>
      .
    </p>
  </div>
  <p>
    Define user-related{' '}
    <a href="/docs/agents/manage-apm-agents/agent-metrics/agent-attributes">
      custom attributes
    </a>{' '}
    to associate with a Browser page view (user name, account name, and product
    name). The values are automatically associated with pre-existing keys (
    <code>user</code>, <code>account</code>, and <code>product</code>), then
    attached to the parent APM transaction. You can also{' '}
    <a href="/docs/insights/new-relic-insights/decorating-events/insights-custom-attributes#forwarding-attributes">
      attach (or "forward") these attributes
    </a>{' '}
    to Browser{' '}
    <a href="/docs/agents/manage-apm-agents/agent-metrics/agent-attributes#destinations">
      PageView
    </a>{' '}
    events.
  </p>
  <h2>Parameters</h2>
  <table>
    <thead>
      <tr>
        <th width="25%">Parameter</th>
        <th>Description</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>
          <p>
            <code>$user_value</code>
          </p>
          <p>
            <em>string</em>
          </p>
        </td>
        <td>
          Required (can be null). Specify a name or username to associate with
          this page view. This value is assigned to the <code>user</code> key.
        </td>
      </tr>
      <tr>
        <td>
          <p>
            <code>$account_value</code>
          </p>
          <p>
            <em>string </em>
          </p>
        </td>
        <td>
          <p>
            Required (can be null). Specify the name of a user account to
            associate with this page view. This value is assigned to the{' '}
            <code>account</code> key.
          </p>
        </td>
      </tr>
      <tr>
        <td>
          <p>
            <code>$product_value</code>
          </p>
          <p>
            <em>string</em>
          </p>
        </td>
        <td>
          <p>
            Required (can be null). Specify the name of a product to associate
            with this page view. This value is assigned to the{' '}
            <code>product</code> key.
          </p>
        </td>
      </tr>
    </tbody>
  </table>
  <h2>Examples</h2>
  <h3 id="report-three">Record three user attributes</h3>
  <pre>
    NewRelic.Api.Agent.NewRelic.SetUserParameters("MyUserName", "MyAccountName",
    "MyProductName");{'\n'}
  </pre>
  <h3 id="report-two">Record two user attributes and one empty attribute</h3>
  <pre>
    NewRelic.Api.Agent.NewRelic.SetUserParameters("MyUserName", "",
    "MyProductName");{'\n'}
  </pre>
