## Use the query builder: basic and advanced modes [#specify-data]

The query builder has several modes:

* [Basic mode](/docs/chart-builder/use-chart-builder/choose-data/use-basic-mode-specify-data)
* [Advanced (NRQL) mode](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data)
* [Advanced (PromQL-style) mode](#chart-builder-promql)

<Callout variant="tip">
  You can also switch to the **[Data explorer](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer)** tab at any time to browse your data and create charts visually, without performing queries.
</Callout>

<CollapserGroup>
  <Collapser
    id="chart-builder-basic"
    title="Basic mode"
  >
    Basic mode doesn't require a query language, and so it's useful if you aren't familiar with [NRQL](/docs/query-data/nrql-new-relic-query-language/getting-started/introduction-nrql) or [SQL](https://en.wikipedia.org/wiki/SQL). It's also a quick way to answer one-off questions, such as how many cities outside of the United States are accessing the shopping site for your company.

    ![querybuilder04.png](./images/querybuilder04.png "querybuilder04.png")

    <figcaption>
      **[one.newrelic.com](https://one.newrelic.com) > Query your data > Query builder >** **Basic mode** Use basic mode to guide you through the process to specify data for a chart.
    </figcaption>

    With **basic mode** you can:

    * Choose your data source. In basic mode, you start by selecting an event and then picking the attribute that corresponds to the information to show in your chart.
    * Specify (filter) the data for your chart. You can narrow the results and choose an attribute to facet by.
    * Select the time range to present in the chart. You can select a time range from the menu or use the custom option to create your own time range.
  </Collapser>

  <dd>
    See [Use basic mode](/docs/chart-builder/use-chart-builder/choose-data/use-basic-mode-specify-data) for more information about this feature.
  </dd>

  <Collapser
    id="chart-builder-advanced"
    title="Advanced (NRQL) mode"
  >
    With the **advanced (NRQL) mode** you can create your own queries using the [NRQL query language](https://docs.newrelic.com/docs/insights/nrql-new-relic-query-language/using-nrql/introduction-nrql). NRQL queries may be simple or complex, depending on what data you want to use and how to display it in your chart. This example of the advanced mode shows the NRQL query with the same data as used in basic mode.

    ![querybuilder05.png](./images/querybuilder05.png "querybuilder05.png")

    <figcaption>
      **[one.newrelic.com](https://one.newrelic.com) > Query your data > Query builder > Advanced (NRQL)**. Use advanced (NRQL) mode to write a query that specifies data for a chart.
    </figcaption>

    <Callout variant="tip">
      With APM's [real time streaming](/docs/agents/manage-apm-agents/agent-data/real-time-streaming), you can query and visualize your data for transactions, errors, and custom events in dashboards.
    </Callout>

    See [Use advanced (NRQL) mode](/docs/chart-builder/use-chart-builder/choose-data/use-advanced-nrql-mode-specify-data) for more information about using this feature.
  </Collapser>

  <Collapser
    id="chart-builder-promql"
    title="Advanced (PromQL-style) mode"
  >
    With the **advanced (PromQL-style) mode** you can [run basic PromQL-style queries](/docs/query-your-data/explore-query-data/chart-builder/use-advanced-promql-mode-specify-data).
  </Collapser>
</CollapserGroup>

You can switch between basic and advanced modes while defining your query in order to:

* customize some of the values.
* learn how NRQL queries are structured using the data you specified in basic mode.

<Callout variant="important">
  If you build your query in basic mode and update it using advanced mode, you **cannot** return to basic mode to edit that query. You must make any additional changes in advanced (NRQL) mode.
</Callout>

Once you specify the data you want, your chart is ready to view. Your query automatically runs in basic mode as you make changes to the inputs. If using advanced (NRQL or PromQL-style) mode, run your query.
