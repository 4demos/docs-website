  <h3>New</h3>
  <ul>
    {' '}
    <li>
      The AWS Lambda integration can now be monitored in these additional
      regions: South America (Sao Paulo/sa-east-1), Asia Pacific
      (Mumbai/ap-south-1), Canada (Central/ca-central-1) and EU
      (London/eu-west-2).
    </li>
  </ul>
  <h3>Changes</h3>
  <ul>
    {' '}
    <li>
      The descriptions of DynamoDB table, region and global secondary index
      entity types have been updated in the{' '}
      <a href="https://docs.newrelic.com/docs/integrations/amazon-integrations/aws-integrations-list/aws-dynamodb-monitoring-integration">
        AWS DynamoDB monitoring integration
      </a>{' '}
      document.
    </li>{' '}
    <li>
      The list of specific permissions that are required to fetch data from AWS
      for each integration has been updated in the{' '}
      <a href="http://docs.newrelic.com/docs/integrations/amazon-integrations/getting-started/integrations-managed-policies">
        Integrations and managed policies
      </a>{' '}
      document. Note that New Relic only recommends using a custom policy only
      if you're unable to use the{' '}
      <a href="http://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_managed-vs-inline.html#aws-managed-policies">
        AWS <code>ReadOnlyAccess</code> managed policy
      </a>
      .
    </li>
  </ul>
