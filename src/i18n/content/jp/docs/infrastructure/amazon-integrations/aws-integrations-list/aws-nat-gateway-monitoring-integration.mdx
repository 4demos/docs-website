当社の AWS ネットワーク アドレス変換 (NAT) ゲートウェイ統合は、NAT ゲートウェイ サービスから New Relic アカウントにデータをレポートします。

## 特徴 [#features]

この統合により、 [NAT ゲートウェイ](https://docs.aws.amazon.com/vpc/latest/userguide/vpc-nat-gateway.html) のテレメトリ データを New Relic で確認できます。サービスを監視し、受信データをクエリし、ダッシュボードを構築して、すべてを一目で確認できます。

## 統合をアクティブ化する [#activate]

この統合を有効にするには、 [AWS サービスを New Relic に接続する手順に](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure)従ってください。

## 設定とデータ転送 [#configure]

データの報告には 2 つのオプションがあります。 [ストリーミング データの](/docs/apis/nerdgraph/examples/nerdgraph-streaming-export) 使用をお勧めしますが、 [ポーリング](/docs/infrastructure/infrastructure-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations) もサポートされています。

ポーリングを選択した場合、 [構成オプション](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations)を使用してポーリング頻度を調整できます。AWS 統合の[デフォルトのポーリング情報](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) :

* New Relicのポーリング間隔：5分
* Amazon CloudWatchのデータ間隔：1分または5分

## データを見つけて使用する [#find-use-data]

この統合からのデータは、 [`Metric` データ型](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics)に添付されます。

データを見つける 1 つの方法: **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Metrics and events** に移動し、 `aws.natgateway`でフィルターします。

## データの使用に関するヒント [#tips]

データのカスタム グラフを作成する方法については、 [NRQL ドキュメント を](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language)参照してください。この種のデータのクエリに関するヒントについては、 [メトリック データのクエリ](/docs/data-apis/understand-data/metric-data/query-metric-data-type)を参照してください。

データの変更に関する通知を受け取るアラートを設定できます。[アラートの作成の詳細については、こちらをご覧ください](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts)。
