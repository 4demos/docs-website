通知は、Slack、Jira、ServiceNow、メールなどのサードパーティサービスに通知イベントを送信するさまざまな方法を統合したものです。また、Webhookを使って、互換性のあるサードパーティサービスにデータを送信することもできます。

宛先やメッセージテンプレートを設定することで、お客様のシステムとの連携が可能です。

## 目的地 [#destinations]

宛先は、サードパーティ システムの一意の識別子と接続の詳細です。宛先を使用して、New Relic とサードパーティ システム間で統合、通知の送信、データの共有を行います。

## メッセージ テンプレート (チャネル) [#message-templates]

メッセージ テンプレート、またはチャネルは、ワークフローやその他の New Relic エンティティによってトリガーされる通知イベントを送信するために必要なすべての情報を含む構成オブジェクトです。「送信先」と「プロパティ」は、メッセージ テンプレートの主要な構成要素です。メッセージ テンプレートを使用すると、サード パーティの宛先に送信する前に、通知イベント データをカスタマイズできます。

## API [#api]

NerdGraph APIを使用して、[宛先](/docs/apis/nerdgraph/examples/nerdgraph-api-notifications-destinations)と[通知チャネル](/docs/apis/nerdgraph/examples/nerdgraph-api-notifications-channels)を表示および構成することもできます。

## セキュリティ [#security]

送信先と通知チャネルに使用される認証情報は、 [New Relic セキュリティ基準](/docs/security/security-privacy/compliance/data-encryption)に従って安全に保存されます。
