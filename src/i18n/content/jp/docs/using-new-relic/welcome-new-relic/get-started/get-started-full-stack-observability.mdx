真のオブザーバビリティにより、デジタルシステム全体で起こっていることやその理由を、どのようなソリューションを使用していてもいつでも知ることができます。全体像を把握し、それにより、クラウドでマイクロサービスを実行しているコンテナやモバイルウェブサイトのショッピングカートのボタンなど、アプリケーションとデバイスで価値を生むことが可能になります。

当社のプラットフォームは、オブザーバビリティを提供することで、単なるモニタリングにとどまらず、お客様の環境をプロアクティブかつ予測的に改善するためのデータの洞察を提供します。

## データをNew Relicに投入 [#integrations]

当社は、データをNew Relicに取り込み、すべてを1か所で分析できるようにするための、[数百のツール](https://newrelic.com/instant-observability/)を提供します。当社のソリューションは、問題を迅速に解決し、複雑なシステムを維持しながらコードを改善し、デジタル変革を加速させるのに役立つツールを提供します。

報告や分析が必要なデータの種類にかかわらず、当社は[お客様に合わせたソリューション](https://newrelic.com/instant-observability)をご用意しています。ただし、状況に応じて構築済みのソリューションを提供しない場合は、[お客様独自のソリューションを構築するための強力なオプション](/docs/data-apis/custom-data/intro-custom-data)も用意しています。

## すべての回答を1か所に [#answers]

データが[New Relic](/docs/using-new-relic/welcome-new-relic/get-started/introduction-new-relic)に入ると、当社のツールを備えたUIにより、システムを取り巻く何層もの複雑さに対処できます。これはすべて1つのプラットフォーム上にあるため、診断アプリケーションを切り替える必要はありません。

パターンの条件を指定してデータを検索し、[データプラットフォーム](/docs/data-ingest-apis/get-data-new-relic/getting-started/get-started-telemetry-data-platform)を使用してデータを発見し、または[機械学習ツール](/docs/new-relic-one/use-new-relic-one/new-relic-ai/introduction-new-relic-ai)から積極的データを取得できます。

[フルプラットフォームユーザー](/docs/users-roles#user-type)として、オブザーバビリティツール全体にアクセスできます。すべてのツールは相互につながっており、[New Relic One](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one)でアクセスできます。

エージェントやインテグレーションを通じてNew Relicに投入するすべてのデータは、プラットフォームの分析やモニター機能に入力する[メトリクス、イベント、ログおよびトレース](https://newrelic.com/platform/telemetry-data-101/)です。New Relicはデータを意味のある方法で関連付けるため、[データを詳しく調べ](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer)、[ダッシュボードを構築し](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards)、[アラートを設定](/docs/alerts)できます。

すぐに使用できるオブザーバビリティUI体験により、ソフトウェアスタック全体を一元化されたプラットフォームで容易に可視化、解析およびトラブルシュートできます。[New Relicエクスプローラー](/docs/new-relic-one/use-new-relic-one/core-concepts/entity-explorer-view-performance-across-apps-services-hosts)は、パフォーマンスの傾向と問題を簡単に検知できるように、システムにあるすべてのエンティティと接続方法を1か所に統合します。インフラストラクチャの状態をアプリケーションのパフォーマンスやエンドユーザーの行動に自動的に接続することで、ノイズをカットして有効な信号を見つけることができます。

## スタック内のどこからでもトラブルシューティングが可能 [#start-anywhere]

完全につながっていることで、New Relic UIを使用してオブザーバビリティをスタックのあらゆる要素から開始できます。たとえば、問題のあるKubernetesポッドで実行中のアプリケーションのトレースから、重要なインフラストラクチャログにアクセスできます。

[New Relic One](/docs/new-relic-one/use-new-relic-one/core-concepts/new-relic-explorer-view-performance-across-apps-services-hosts/)の[Explorer](https://one.newrelic.com/)を使用すると、ソフトウェアの全スタックにアクセスして観察したり、パフォーマンスデータとアラートステータスを一目で確認して関係を把握できます。シンプルで強力なビジュアルツールにより、すべての[エンティティ](/docs/new-relic-one/use-new-relic-one/core-concepts/what-entity-new-relic/)、つまり、データをレポートする識別可能な任意の対象を監視できます。New Relicエコシステムでは、エンティティは、アプリケーション、ホスト、コンテナ、データベースサービスなどの基本コンポーネントを含みますが、こうした要素のカスタムグループを表す場合もあります。[エンティティは独自に作成](https://github.com/newrelic/entity-definitions#entity-definitions)することもできます。インストゥルメントするエンティティが増えるほど、投入するデータが増えます。New Relicに投入したデータが多いほど、メトリクスやイベント、ログ、トレースについての理解が深まります。
