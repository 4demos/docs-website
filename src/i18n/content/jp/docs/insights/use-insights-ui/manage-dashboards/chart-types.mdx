[New Relic One](/docs/new-relic-one/use-new-relic-one/get-started/introduction-new-relic-one)は、あなたのニーズに最も適した形式でデータを表現できる、様々な視覚化をサポートしています。チャートのリフレッシュレートは、クエリのタイムフレームを基に算出されます。

## イベントのチャートタイプ [#widget-types]

New Relic Oneの以下のオプションを使用すると、各自のデータを参照できます。

* [データエクスプローラー](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer)
* 基本または詳細（NRQL）クエリ
* 詳細PromQLスタイルモード

[New Relic Oneクエリビルダ](/docs/specify-data-chart)でチャートを作成する場合は、クエリに適した任意のチャートタイプを選択できます。クエリで指定されているデータによっては、利用できないチャートタイプもあります（グレー表示）。小テーブルに、InsightsおよびNew Relic Oneのチャートタイプ別の利用可能性について示しています。

    **ビルボード**チャートは、結果を単一の非常に見やすい値で表します。これは、次のクエリのように、クエリが単一の数を返す場合に利用できます:

    ```
    SELECT uniqueCount(session) FROM PageView since 1 day ago
    ```

    ビルボードチャートは、 [Apdex](/docs/apm/new-relic-apm/apdex/apdex-measure-user-satisfaction)関数のようなsinglestatのクエリに便利です。**ビルボード** を利用して、値を一目でモニターできます。

    ビルボードは、複数の状況にまたがる単一の値を比較する際にも使用できます。たとえば、[`COMPARE WITH`](/docs/insights/nrql-new-relic-query-language/nrql-reference/nrql-syntax-components-functions#sel-compare)句を使用して、2つの異なる時間枠の同じ属性を比較します。

    ![chart_billboard.png](./images/chart_billboard.png "chart_billboard.png")

    <figcaption>
      **ビルボード**チャートは、単一の値を大きく読みやすいフォントで表示します。
    </figcaption>

    データをビルボードチャートで利用できるようにするには、単一の値を検索します。ドロップダウンからファセットを追加することで、複数の値を表示できます。

    ビルボードでは**Warning（警告）**と**Critical（重要）**の閾値を設定でき、チャートをそれぞれ黄色または赤色でハイライト表示します。閾値フィールドは、チャート編集ペインの右側にあり、それぞれ黄色と赤色でハイライト表示されています。

    値が所定の値を上回るのでなく下回った場合にチャートを強調表示したい場合、**Warning（警告）**閾値を**Critical（危険）**閾値より高く設定します。

    <Callout variant="tip">
      閾値フィールドは、単一の値を表すダッシュボードにおいてのみ利用できます。たとえば、`COMPARE WITH`句を使用すると、編集ペインの閾値設定が無効になります。
    </Callout>

    **ブレット**チャートタイプを使用して、リミットと呼ばれるターゲットまたは目標に向けた進捗状況を表します。このチャートタイプを使用するには、次のような単一の数を返すクエリを実行します:


    ![chart_bullet.png](./images/chart_bullet.png "chart_bullet.png")

    <figcaption>
      **ブレットチャート**は、リミットと比較した値を表示します。

    <Callout variant="tip">
      **ブレット**チャートタイプは、単一の値を表すチャートにのみ使用できます。たとえば、[`COMPARE WITH`](/docs/insights/nrql-new-relic-query-language/nrql-reference/nrql-syntax-components-functions#sel-compare)を使用するクエリでは、オプションとしてブレットがリストされません。

    特定の目標に向けたステップとなる一連の関連イベントを評価するには、[ファネル](/docs/insights/nrql-new-relic-query-language/nrql-query-examples/funnels-evaluate-data-series-events)を使用します。ファネルでは、以下を問い合わせることができます: 「（A）を行った人のうち、（B）も行ったのは何人であり、また（A）と（B）を行った人のうち、（C）も完了した人は何人だったでしょうか？」

    ![temp-funnel-chart.png](./images/temp-funnel-chart.png "ファネルチャートのサンプル")

    <figcaption>
      ショッピングサイトに関するサンプルの**ファネル**チャート。
    </figcaption>

    この例では、ランディングページを見た訪問者のうち、何人が以下を行ったか判別する必要があります：

    * 製品を閲覧した。
    * 次に、買い物かごに1つ以上の製品を追加した。
    * 最後に、買い物かご内の商品を購入した。

    他のクエリとは異なり、このクエリには[`SELECT`](https://docs.newrelic.co.jp/docs/insights/nrql-new-relic-query-language/nrql-reference/nrql-syntax-components-functions#state-select) [`funnel()`](/docs/insights/nrql-new-relic-query-language/nrql-reference/nrql-syntax-components-functions#func-funnel)ステートメントを使用する必要があります。ファネルチャートタイプは、他のどのクエリの種類からも選択できません。

    また、ファネルチャートタイプは、ステップ数が8以下のクエリでしか使用できません。9以上のステップを含むファネルクエリは、JSONとしてのみ結果を返します。

    ```
    SELECT funnel(session,
        WHERE pageUrl like '%/index.html' AS 'Landing',
        WHERE pageUrl LIKE '%/browse/phones' AS 'Browse ',
        WHERE pageUrl LIKE '%/shoppingcart' AS 'Cart',
        WHERE pageUrl LIKE '%/checkout' AS 'Purchased')
        FROM PageView where appName = 'WebPortal' AND duration < 1
        SINCE 1 week ago
    ```

## メトリックデータ向けチャートタイプ [#metric-widget-types]

メトリックチャートは、他のイベントベースのチャートとは別に固有のものです。これは、ご利用のエージェントからNew Relicに報告される、集計メトリックデータを表示できます。Insightsダッシュボードで[メトリックチャートを追加してカスタマイズ](/docs/insights/new-relic-insights/managing-dashboards-data/add-customize-metric-widgets)するには、[メトリックエクスプローラー](/docs/insights/new-relic-insights/explore/metric-data-explorer-search-chart-metrics-sent-new-relic-agents)を使用します。

New Relic Oneでは、まだメトリックエクスプローラーに対応していません。ただし、ダッシュボードにメトリックチャートを追加した後は、クエリビルダーを**詳細（NRQL）モード**で使用してクエリを編集できます。

New Relic Oneで詳細（NRQL）モードを使用するには：

1. [one.newrelic.com](https://one.newrelic.com)に移動します。
2. 「データのクエリ」アイコンをクリックします。
3. クエリビルダーが**基本**モードで開いた場合、基本モードコンテナの右上から**詳細 (NRQL)**をクリックしてください。
4. 上級（NRQL）モードで**My recent queries**ドロップダウンを使用して、Insightsダッシュボードのメトリックチャートに関連したクエリを選択します。

クエリビルダーを開くと、最後に使用したクエリのモードが基本モードまたは詳細（NRQL）モードで表示されます。上級（NRQL）モードの**My recent queries**ドロップダウンでは、InsightsとNew Relic Oneの両方の基本モードと上級（NRQL）モードのNew Relicクエリがすべて表示されます。
