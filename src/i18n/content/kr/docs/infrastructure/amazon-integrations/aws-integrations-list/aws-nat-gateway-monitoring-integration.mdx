AWS NAT(Network Address Translation) 게이트웨이 통합은 NAT 게이트웨이 서비스에서 New Relic 계정으로 데이터를 보고합니다.

## 특징 [#features]

이 통합을 통해 New Relic에서 [NAT 게이트웨이](https://docs.aws.amazon.com/vpc/latest/userguide/vpc-nat-gateway.html) 원격 측정 데이터를 볼 수 있습니다. 서비스를 모니터링하고, 들어오는 데이터를 쿼리하고, 대시보드를 구축하여 모든 것을 한눈에 관찰할 수 있습니다.

## 통합 활성화 [#activate]

이 통합을 활성화하려면 [AWS 서비스를 New Relic에 연결하기 위한 지침을](/docs/infrastructure/infrastructure-integrations/getting-started/connect-aws-integrations-infrastructure)따르십시오.

## 구성 및 데이터 전송 [#configure]

데이터 보고에는 두 가지 옵션이 있습니다. [스트리밍 데이터를](/docs/apis/nerdgraph/examples/nerdgraph-streaming-export) 사용하는 것이 좋지만 [폴링](/docs/infrastructure/infrastructure-integrations/cloud-integrations/configure-polling-frequency-data-collection-cloud-integrations) 도 지원됩니다.

폴링을 선택하면 [구성 옵션을](/docs/integrations/new-relic-integrations/getting-started/configure-polling-frequency-data-collection-cloud-integrations)사용하여 폴링 빈도를 조정할 수 있습니다. AWS 통합에 대한[기본 폴링 정보](/docs/infrastructure/amazon-integrations/aws-integrations-list/aws-polling-intervals-infrastructure-integrations) :

* New Relic 폴링 간격: 5분
* Amazon CloudWatch 데이터 간격: 1분 또는 5분

## 데이터 찾기 및 사용 [#find-use-data]

이 통합의 데이터는 [`Metric` 데이터 유형](/docs/data-apis/understand-data/new-relic-data-types/#dimensional-metrics)에 연결됩니다.

데이터를 찾는 한 가지 방법: **[one.newrelic.com](https://one.newrelic.com/all-capabilities) > Metrics and events** 로 이동하고 `aws.natgateway`로 필터링합니다.

## 데이터 사용 팁 [#tips]

데이터의 사용자 지정 차트를 만드는 방법은 [NRQL 문서를](/docs/query-your-data/nrql-new-relic-query-language/get-started/introduction-nrql-new-relics-query-language)참조하세요. 이러한 종류의 데이터 쿼리에 대한 팁은 [메트릭 데이터 쿼리를](/docs/data-apis/understand-data/metric-data/query-metric-data-type)참조하세요.

데이터 변경 사항에 대한 알림을 받도록 알림을 설정할 수 있습니다. [알림 만들기에 대해 자세히 알아보세요](/docs/alerts-applied-intelligence/new-relic-alerts/learn-alerts/introduction-alerts).
